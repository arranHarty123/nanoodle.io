Decentralise Nano

Transactions on the Nano network are voted on to ensure they are legitimate and protect the network against bad actors. This voting is conducted by a Nano node, software that can be run by anybody wishing to participate in the network. See here for details on how to run a Nano node and help decentralise the nano network.

Nano nodes are associated with 1 or more accounts (within a wallet), all of which can be nominated as a Representative by other accounts. This is an efficient way for accounts to ensure that they can participate in the network voting, even if they aren't going to be consistenly online; a representative will vote on behalf of all accounts that have nominated the account. The ability for an account to change its nominated representative is an important feature for decentralisation of the Nano network.

A representative's voting power is related to the amount of Nano that is assigned to the representative from accounts. In many cases, individuals will keep a default representative on their account and this can result in a more centralised Nano network. To support Nano decentralisation, individuals should change their representative based on an assessment of:

  • Node Availability
  • Reputation
  • Diversity
  • Version
  • Processing Power
  • Voting Weight
  • Synchronization

Increasing the number of good representatives improves network health.
